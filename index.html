"In che sequenza risolvo?"

Il Metodo SPAZIO: Come Pensare in DP

🧠 SPAZIO è l'acronimo che ti guiderà nella risoluzione di qualsiasi problema DP:

  • S - Stati: "Cosa Caratterizza la Mia Situazione?"
  • P - Passaggi: "Come Transito tra Stati?"
  • A - Assiomi: "Quali Sono i Casi Base?"
  • Z - Zona: "In Che Ordine Calcolo?"
  • I - Implementa: "Come Traduco in Codice?"
  • O - Ottimizza: "Posso Fare Meglio?"

🌟 S - Stati: "Cosa Caratterizza la Mia Situazione?"

PROCESSO MENTALE GUIDATO:

1. "Cosa sto processando?"

  • → Sequenza: T[i] = primi i elementi
  • → Matrice: T[i][j] = sottomatrice fino a (i,j)
  • → Grafo: T[nodo] = soluzione fino a nodo

2. "Cosa devo ricordare del passato?"

  • → Ultimo elemento? T[i][ultimo]
  • → Risorse usate? T[i][peso_usato]
  • → Obiettivi raggiunti? T[i][ha_visto_012]
ESEMPIO VISIVO: Problema: "Stringhe senza 11" ↓ "Devo sapere se l'ultimo era 1!" ↓ Stati: T[i][0] = finisce con 0 T[i][1] = finisce con 1

🌟 P - Passaggi: "Come Transito tra Stati?"

LA DOMANDA D'ORO:
"Se sono nello stato X, quali stati posso raggiungere?"

VISUALIZZAZIONE: Stato[i-1][0] ━━━━━┓ ┣━━> Stato[i][0] Stato[i-1][1] ━━━━━┛ ┗━━> Stato[i][1] PATTERN COMUNI: 1. SCELTA: prendo/non prendo T[i] = max(T[i-1], T[i-2] + val[i]) 2. SOMMA: tutti i modi T[i] = Σ T[j] per j validi 3. ESTREMO: migliore opzione T[i] = min/max(T[j] + costo(j→i))

🌟 A - Assiomi: "Quali Sono i Casi Base?"

MENTALITÀ:
"Qual è il problema più piccolo che so risolvere?"

TRUCCHI VISIVI:

  • Sequenze: T[0] = ? Di solito: vuoto = 1 modo
  • Matrici: T[0][0] = ? Di solito: cella start = 1
  • Conteggi: T[0][x] = ? Di solito: 0 ovunque tranne stato init

Template Universale DP 
# TEMPLATE UNIVERSALE DP
def risolvi_dp(input):
    # 1. INIZIALIZZA stati
    dp = crea_tabella()
    
    # 2. CASI BASE (Assiomi)
    inizializza_casi_base(dp)
    
    # 3. RIEMPI seguendo l'ordine (Zona)
    for stato in ordine_corretto():
        # 4. APPLICA transizione (Passaggi)
        for precedente in stati_precedenti_validi(stato):
            dp[stato] = aggiorna(dp[stato], dp[precedente])
    
    # 5. RITORNA soluzione
    return dp[stato_finale]

🎨 CAPITOLO 4: PATTERN VISIVI - Riconoscere al Volo

I 7 Pattern che Coprono il 90% dei Problemi

📊 Pattern 1: Fibonacci Nascosto

SEGNALI: "dipende da 2 precedenti" + "somma/scelta"

VISUALIZZAZIONE: • / \ • • ← Struttura ricorsiva / \ / \ • • • • ESEMPIO CLASSICO: "Scale con salti 1-2" = Fibonacci! T[i] = T[i-1] + T[i-2]

🔍 Riconosci il Pattern: Quale di questi è Fibonacci nascosto?

Tile 2×n
Modi di riempire rettangolo con tiles 2×1
Ordinamento
Ordinare array in ordine crescente

📊 Pattern 2: Zaino (Knapsack)

SEGNALI: "scegli oggetti" + "capacità limitata"

GRIGLIA MENTALE: peso → o ┌─┬─┬─┬─┬─┐ g │0│0│0│v│v│ v = valore g ├─┼─┼─┼─┼─┤ oggetto e │0│0│0│v│v│ t ├─┼─┼─┼─┼─┤ t │0│w│w│V│V│ V = valore i └─┴─┴─┴─┴─┘ migliore ↓ T[i][w] = max(T[i-1][w], T[i-1][w-peso[i]] + valore[i])

📊 Pattern 3: LCS/LIS (Sottosequenze)

SEGNALI: "sottosequenza" + "proprietà da mantenere"

PROCESSO MENTALE: Sequenza: 3 1 4 1 5 ↓ ↓ ↓ ↓ ↓ Prendo? N S S N S → 1,4,5 crescente! T[i] = max(T[j] + 1) per j

📊 Pattern 4: Intervalli

SEGNALI: "substring" + "ottimizza su range"

TABELLA TRIANGOLARE: j → i ┌─┬─┬─┬─┐ ↓ │X│ │ │ │ Diagonale = stringhe lunghe 1 ├─┼─┼─┼─┤ │X│X│ │ │ Sopra = stringhe lunghe 2 ├─┼─┼─┼─┤ │X│X│X│ │ Ancora sopra = lunghe 3 └─┴─┴─┴─┘ T[i][j] = funzione(T[i+1][j], T[i][j-1], T[i+1][j-1])

📊 Pattern 5: Alberi/DAG

SEGNALI: "percorsi" + "grafo aciclico"

APPROCCIO:
Per ogni nodo in ordine topologico:
  T[nodo] = migliore tra:
    T[pred] + costo(pred→nodo)
    per ogni predecessore

📊 Pattern 6: Matrici Cumulative

SEGNALI: "somma in sottomatrice" + "query multiple"

PREPROCESSING: ┌─┬─┬─┐ ┌─┬──┬──┐ │1│2│3│ │1│3 │6 │ ├─┼─┼─┤ → ├─┼──┼──┤ │4│5│6│ │5│12│21│ └─┴─┴─┘ └─┴──┴──┘ originale cumulativa Somma(r1,c1,r2,c2) = O(1)!

📊 Pattern 7: Stati Multipli

SEGNALI: "vincoli complessi" + "storia importante"

DIMENSIONI EXTRA: T[pos][ultimo][conteggio][flag] ↓ ↓ ↓ ↓ dove quale quanti proprietà sono elemento ne ho speciali Esempio: "esattamente k elementi diversi" T[i][mask][k] = fino a i, elementi usati, conteggio

🎭 Trucco 2: "Stati Impossibili"

❌ NOVIZIO: calcola tutto
for i in range(n):
    for j in range(m):
        dp[i][j] = calcola()
        
# Calcola n×m stati
✅ ESPERTO: salta impossibili
for i in range(n):
    for j in range(min(i+1, m)):  # j ≤ i
        if è_possibile(i, j):
            dp[i][j] = calcola()
            
# Risparmio: fino a 50% del tempo!

🎭 Trucco 3: "La Ricorsione con Memoria"

Quando bottom-up è complicato, usa top-down!

Template Top-Down Magico 
from functools import lru_cache

@lru_cache(maxsize=None)
def dp(stato):
    # Caso base
    if stato == base:
        return valore_base
    
    # Ricorsione
    risultato = neutro
    for prossimo in successori(stato):
        risultato = combina(risultato, dp(prossimo))
    
    return risultato

# Magia: calcola solo stati raggiungibili!

🪄 MAGIA: Questo pattern funziona quando:

  • ✅ Non tutti gli stati sono raggiungibili
  • ✅ L'ordine di calcolo è complicato
  • ✅ Vuoi scrivere codice più leggibile

🎭 Trucco 4: "Rolling Array Pattern"

Ottimizzazione di memoria quando serve solo "window" di stati

PRIMA: O(n×m) memoria
dp = [[0]*m for _ in range(n)]

for i in range(n):
    for j in range(m):
        dp[i][j] = funzione(
            dp[i-1][j],
            dp[i][j-1]
        )
DOPO: O(m) memoria
prev = [0] * m
curr = [0] * m

for i in range(n):
    for j in range(m):
        curr[j] = funzione(
            prev[j],
            curr[j-1]
        )
    prev, curr = curr, prev

🎭 Trucco 5: "Coordinate Compression"

Quando lo spazio è enorme ma i valori sono pochi

SCENARIO: "DP su range [1, 10^9] ma solo 1000 valori distinti"

❌ INGENUO: 
dp = [0] * (10**9)  # CRASH!
✅ COMPRESSO: 
valori_unici = sorted(set(input))
mappa = {v: i for i, v in enumerate(valori_unici)}
dp = [0] * len(valori_unici)
